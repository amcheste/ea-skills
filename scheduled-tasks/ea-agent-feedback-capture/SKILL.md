You are running a nightly feedback capture task for Alan Chester's ea-agent project. Your job is to review today's Cowork session transcripts, extract any improvements to the ea-agent that were discovered in conversation, and — if anything is worth capturing — create a feature branch, commit it locally, then use the Mac shell to push the branch and open a GitHub PR.

## Step 1: Review Today's Sessions

Use the session listing tool to get a list of recent sessions. Then read the transcripts of any sessions from today (or the last 24 hours). Focus on finding:

1. **Prompt / skill improvements** — Any prompts, instructions, or skill definitions that were discussed, refined, or found to work better in practice. This includes changes to how skills are worded, new steps added to a skill, or better default behaviors.
2. **Bug / behavior fixes** — Anything the ea-agent did wrong, inconsistently, or that Alan corrected or expressed frustration about. Include implied bugs (e.g. "it should have done X but did Y").

Ignore anything unrelated to the ea-agent (e.g. discussions about SalienceOS, research papers, or other projects).

If you find nothing relevant, stop here and output: "No ea-agent improvements found in today's sessions. No PR created."

## Step 2: Summarize the Improvements

For each improvement found, write a structured entry:

```
### [Type: Prompt Improvement | Bug Fix | New Skill]
**Skill affected:** <skill name, or "general" if not skill-specific>
**Discovered in:** <brief description of the conversation context>
**Improvement:** <what should change and why>
**Suggested change:** <concrete description of what to update in the repo>
```

## Step 3: Create a Feature Branch and Write Improvements File

If there are improvements:

1. Go to the ea-agent repo: `~/Repos/amcheste/ea-agent/`
2. Make sure you're on the latest `develop` branch: `git checkout develop && git pull`
3. Create a feature branch with today's date: `git checkout -b feedback/YYYY-MM-DD-session-improvements`
4. Create a new file at `improvements/YYYY-MM-DD-session-improvements.md` with the following structure:

```markdown
# Session Improvements — YYYY-MM-DD

> Auto-captured from Cowork session transcripts on YYYY-MM-DD.

## Summary
<1-2 sentence summary of what was found>

## Improvements

<list of structured improvement entries from Step 2>

## Action Items
<bullet list of specific files or skills that need updating>
```

5. Stage and commit the file. Note: git identity may not be set in the sandbox — if `git commit` fails with "Author identity unknown", run these first:
   ```
   git config user.email "amcheste@gmail.com"
   git config user.name "Alan Chester"
   ```
   Then commit:
   ```
   git add improvements/YYYY-MM-DD-session-improvements.md
   git commit -m "feat: capture session improvements from YYYY-MM-DD

Auto-captured from Cowork session transcripts.

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"
   ```
   Also note: if git complains about an index.lock file, delete it first: `rm -f .git/index.lock`

## Step 4: Push the Branch via Mac Shell

The sandbox cannot reach GitHub directly (no outbound SSH). Use the `mcp__Control_your_Mac__osascript` tool to run the push on your Mac, which has SSH and `gh` already authenticated:

```applescript
do shell script "cd ~/Repos/amcheste/ea-agent && git push -u origin feedback/YYYY-MM-DD-session-improvements 2>&1"
```

Replace `YYYY-MM-DD` with today's actual date before calling the tool.

If the push fails, capture the error output and skip to the fallback at the end.

## Step 5: Open a GitHub PR via Mac Shell

Use `mcp__Control_your_Mac__osascript` again to create the PR with `gh`, always targeting `develop`:

```applescript
do shell script "cd ~/Repos/amcheste/ea-agent && gh pr create --head feedback/YYYY-MM-DD-session-improvements --base develop --title 'Feedback: Session improvements from YYYY-MM-DD' --body 'Auto-captured improvements from YYYY-MM-DD Cowork sessions.\n\nSee improvements/YYYY-MM-DD-session-improvements.md for full details and action items.\n\n🤖 Auto-generated by the ea-agent feedback capture task' 2>&1"
```

Replace `YYYY-MM-DD` with today's actual date. The output of this command will contain the PR URL — capture it for the Step 6 summary.

If `gh` is not found, try the full path: `/opt/homebrew/bin/gh` or `/usr/local/bin/gh`.

## Step 6: Output a Summary

Print a brief summary to the session:
- How many improvements were found
- Which skills are affected
- The PR URL (from the gh output in Step 5)

## Fallback: If git push or gh pr create fails

If either the push or the PR creation fails (and you have a local commit ready), write a fallback file so nothing is lost:

File: `~/ea-agent-pending-improvements.md`

Include:
- The error message from the failed command
- The branch name
- The full improvements text
- The ready-to-run commands to push and create the PR manually:
  ```
  cd ~/Repos/amcheste/ea-agent
  git push -u origin feedback/YYYY-MM-DD-session-improvements
  gh pr create --head feedback/YYYY-MM-DD-session-improvements --base develop --title "Feedback: Session improvements from YYYY-MM-DD" --body "..."
  ```

## Important constraints
- Never commit directly to `main`, `master`, or `develop`
- Always branch off `develop` and PR back to `develop` using `--base develop` and `--head <branch>`
- Do not merge the PR — leave that to Alan
- The repo path on disk is: `~/Repos/amcheste/ea-agent/` (accessible from the sandbox at `/sessions/*/mnt/achester/Repos/amcheste/ea-agent/`)
- Use Bash tool for local git operations; use `mcp__Control_your_Mac__osascript` for anything that needs to reach GitHub
